# Olist 电商用户分层与策略分析
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

## 1. 项目背景
本项目以巴西 Olist 电商平台真实公开交易数据为基础，模拟从 0 到 1 构建一套**可落地的用户价值分层体系**，并针对不同用户群提出精细化运营策略。目标是展示从数据导入、清洗、分析建模到报告输出及商业建议的完整闭环。

## 2. 数据来源
-   **来源**: Kaggle - Brazilian E-Commerce Public Dataset by Olist (9张CSV表格，包含订单、用户、商品、地理信息等维度)
-   **数据获取**: 原始数据文件未包含在本仓库中。请从 [这里](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce) 下载，并按 `sql_queries/` 目录下的指引导入本地MySQL。

## 3. 分析工具
-   **Python (Pandas)**: 数据预处理，如对地理坐标表进行聚合清洗
-   **MySQL**: 数据存储、多表联查、RFM指标计算
-   **Power BI**: 分析报告制作

## 4. 分析流程
1.  **数据清洗与入库**：用 Python 处理地理数据表的主键冲突。对原始地理数据按邮编分组，对经纬度坐标按出现频率进行加权聚合，使每个邮编对应唯一坐标，然后导入 MySQL。
2.  **探索性数据分析**：对GMV、订单状态、地区分布等进行基础分析，形成初步业务认知。
3.  **RFM模型构建**：定义并计算每位用户的R（最近消费间隔）、F（消费频次）、M（消费总金额）。
4.  **阈值确定与用户分层**：分析M值百分比分布与L型长尾特征，结合业务理解确定分层阈值，将用户划分为X个价值群体。
5.  **策略输出**：针对各层用户进行画像分析，并提出差异化运营建议。

## 5. 关键发现与报告
**

![用户分层画像](images/rfm_profile.png)

-   **用户分布**：消费金额呈典型的L型长尾分布，约4.5%的高价值用户贡献了平台约25%的GMV。
-   **分层策略摘要**：
    -   **重要价值用户**：留存率高、消费力强，策略是提供VIP服务和高价值新品优先权。
    -   **价格敏感型用户**：消费频次高但金额低，策略是定期推送折扣信息和满减活动。
