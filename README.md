# Olist Brazilian E-Commerce Public Dataset
# 用户价值分层与流失归因分析

## 项目简介

本项目基于巴西最大电商平台 Olist 的真实交易数据（2016-2018年），针对平台**约两年跨度内复购率仅3%** 且**约63%销售额来自耐消品**的独特业务特征，设计并实现了一套**基于品类购买周期的动态用户分层模型**，精准锁定高价值流失用户，并通过统计检验与定性分析完成流失归因，最终提出可落地的差异化运营策略。

**核心成果**：从0到1构建动态分层模型 → 锁定293名最高优先级挽回目标（人均830雷亚尔，5.5倍于其余流失用户）→ 归因物流延误为流失关键因素（p<0.001）→ 产出6页Power BI数据报告与完整可复现的SQL/Python代码。

## 分析流程
数据诊断 → 模型设计 → 分层打标 → 价值验证 → 流失归因 → 策略输出


### 1. 数据诊断：为什么标准RFM不能用？

- 平台约两年跨度内**复购率仅3%** → F维度（购买频次）完全失效，几乎所有用户都是1次
- 约**63%销售额来自耐消品**（家具、家电、电脑配件等） → 固定R阈值（如365天）会将大量正常静默的耐消品用户误判为流失
- **若按365天未购即流失的固定阈值，27.4%的用户将被误判为流失**

### 2. 模型设计：构建品类感知的动态分层

**R维度 — 基于品类购买周期的动态活跃度**
- 将73个品类按购买周期划分为**短/中/长**三类
- 短周期（快消品）：活跃≤90天，沉默91-180天，流失>180天
- 中周期（半耐用）：活跃≤180天，沉默181-365天，流失>365天
- 长周期（耐消品）：活跃≤365天，>365天为沉默，**不设流失标签**
  - 理由：两年数据不足以覆盖耐消品的自然更换周期（3-5年+），无法区分“已流失”和“正常静默”

**M维度 — 统一消费金额分层**
- 阈值：100 / 200 / 500 雷亚尔
- 分位点验证：约46.3% / 78.6% / 95.4%
- 四层：低消费 / 中消费 / 高消费 / 超高消费
- 超高消费用户仅占4.6%，贡献25.5%销售额（价值倍数5.59），分层区分度显著

**技术实现**
- 在MySQL中以**CTE视图**完成全量用户打标
- 用户粒度：以 `customer_unique_id` 为单位，基于最近一次购买判定所属周期与R状态

### 3. 核心发现

**发现一：修正“虚假活跃”认知**

表面看活跃用户贡献了55.7%的销售额，但深入品类周期后发现：
- **46.2%** 来自长周期耐用品刚购新客（仅因未超正常消费周期被标为“活跃”）
- **2.0%** 来自短周期快消品活跃用户（真正的高频忠诚用户）
- 盲目对全体“活跃”用户促销，将把大量预算浪费在近期不会复购的耐消品新客身上

**发现二：精准锁定高价值流失用户**

- 完成32个细分群体的横向对比，确认**短周期-流失-超高消费**为最高优先级挽回目标
- 仅 **293人**（占全量0.31%），人均消费 **830雷亚尔**
- 其余流失用户人均仅 **150雷亚尔**，差距达 **5.5倍**
- 经 **Welch t检验**（p **≪ 0.001**）和 **Mann-Whitney U检验**（p **≪ 0.001**）双重验证，差异极显著

**发现三：流失误判大幅收窄**

| 模型口径 | 流失判定规则 | 流失用户占比 |
|----------|-------------|-------------|
| 固定阈值（365天） | 所有用户，>365天未购即流失 | **27.4%** |
| 品类周期动态模型 | 短/中周期判定流失，长周期仅沉默 | **14.4%** |
| **误判率降低** | — | **近一半** |

### 4. 流失归因：物流延误是关键因素

| 检验维度 | P0目标(293人) | 活跃高价值(126人) | 检验方法 | p值 | 结论 |
|----------|---------------|-------------------|----------|-----|------|
| 差评率 | 9.34% | 13.97% | 卡方检验 | 0.19 | 不显著 |
| 平均物流 | **15.5天** | **12.3天** | Welch t | **<0.001** | **极显著** |

- 定量证据：P0目标用户平均物流慢 **3.2天**，统计极显著
- 定性印证：26条差评原文中，**物流相关投诉占近一半(40%以上)**
- 次要因素：发错货、客服失联叠加物流延误，进一步放大负面体验

### 5. 策略建议

| 优先级 | 目标群体 | 人数 | 策略 |
|--------|----------|------|------|
| **P0** | 短周期-流失-超高消费 | 293 | 大额券+短信触达，补偿包邮券 |
| P1 | 短周期-流失-高消费 | 980 | 满减券，批量挽回 |
| P1 | 中周期-流失-超高消费 | 363 | 关联推荐+邮件唤醒 |
| P1 | 短周期-沉默-超高消费 | 91 | 预防性优惠券，防流失 |
| 培育 | 短周期-活跃-超高消费 | 126 | 会员权益强化，提升钱包份额 |
| 不干预 | 长周期超高消费（活跃+沉默） | 2,608 | 仅关联推荐，不打扰 |
| 系统 | 所有超高消费用户 | 4,265 | 订单分配更可靠物流渠道 |

## 数据来源

- **数据集**：[Olist Brazilian E-Commerce Public Dataset](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce) (Kaggle)
- **时间跨度**：2016年9月 ~ 2018年10月
- **分析参考日期**：2018年10月17日（数据集中最后有记录的时间点）
- **分析范围**：仅使用 `order_status = 'delivered'` 的已完成订单，确保分析建立在真实成交行为之上

## 复现说明

1. **获取数据**：从 [Kaggle Olist Dataset](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce) 下载原始CSV文件，放入 `0_data/` 目录
2. **数据预处理**：运行 `0_data/EDA_and_dataset_clean.py` 清洗地理信息表
3. **数据库搭建**：按 `1_sql/` 中编号顺序执行SQL脚本（0→1→2→3→4→5→6）
4. **导出分析数据**：`6_export_queries.sql` 生成 `2_csv/` 中的所有CSV
5. **统计检验**：运行 `2_csv/` 中的两个 `.py` 脚本
6. **可视化报告**：用 Power BI Desktop 打开 `3_powerbi/olist_user_segmentation.pbix`

- 本项目为独立学习与分析作品，非Olist官方产出
- 数据集中评论表存在少量不规范数据，已在 `3_special_handling_olist_order_reviews_dataset.sql` 中处理

- 价值倍数 = 金额占比 ÷ 用户占比；>1 表示该群体人均贡献高于全局平均水平
